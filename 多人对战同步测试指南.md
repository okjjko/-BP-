# 多人对战同步功能测试指南

## 修复内容总结

### 关键问题
选手无法操作是因为**身份分配消息没有被处理**，导致权限检查失败。

### 已修复的问题

1. ✅ **`roomManager.js`** - 添加了对 `identityAssigned` 消息的处理
2. ✅ **`gameStore.js`** - 添加了对 `identityAssigned` 事件的监听
3. ✅ **所有关键操作方法** - 添加了 `syncState()` 调用（7 处）

### 测试场景 1：基本 BP 操作同步

**目标**：验证 ban/pick 操作实时同步

**步骤**：
1. 打开**浏览器 A**（正常模式）
2. 打开**浏览器 B**（隐身模式）- 模拟第二个用户
3. 在浏览器 A 中：选择"多人对战" → "主办方" → 创建房间
4. 在浏览器 B 中：选择"多人对战" → "选手" → 输入邀请码 → 输入 ID "Player1"
5. 主办方开始游戏，输入双方 ID（Player1 和 Player2）
6. 在浏览器 B 中 ban 一个植物

**验证点**：
- ✅ 浏览器 A 应立即显示 ban 结果
- ✅ 浏览器 A 控制台应显示：`[gameStore] 收到状态更新 vX`
- ✅ 浏览器 B 控制台应显示：`[gameStore] 已同步状态 vX`
- ✅ StageIndicator 应显示"已同步"状态（2秒内）

**预期日志**：


## 故障排查

### 问题：仍然没有同步日志

**检查项**：
1. 打开浏览器控制台（F12）
2. 检查是否有 `[gameStore] 收到身份分配消息` 日志
3. 检查 `myAssignedPlayer` 是否被正确设置

**调试方法**：
```javascript
// 在浏览器控制台输入
console.log('roomMode:', store.roomMode)
console.log('myRole:', store.myRole)
console.log('myPlayerName:', store.myPlayerName)
console.log('myAssignedPlayer:', store.myAssignedPlayer)
console.log('isMyTurn:', store.isMyTurn)
```

### 问题：权限检查失败

**症状**：点击操作弹出 alert "现在不是你的回合！"

**可能原因**：
1. 身份分配失败（检查 `myAssignedPlayer` 是否为 null）
2. ID 不匹配（选手输入的 ID 与主办方输入的不一致）

**解决方法**：
- 确保选手输入的 ID 与主办方输入的完全一致
- 重新创建房间并加入

### 问题：WebRTC 连接失败

**症状**：
- 无法加入房间
- 控制台显示连接错误

**检查项**：
1. 确保两个浏览器使用相同的网络
2. 检查防火墙设置
3. 检查 PeerJS 信令服务器是否可访问
