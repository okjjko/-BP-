# 多人对战同步诊断指南

## 问题描述
在其中一个客户端上BP时，禁用/选择植物后，状态没有自动同步到其他客户端。

## 诊断步骤

### 1️⃣ 打开浏览器控制台
在所有三个客户端上打开开发者工具（F12），切换到 Console 标签。

### 2️⃣ 检查 WebRTC 连接状态

在每个客户端的控制台中输入以下命令：

```javascript
// 检查连接状态
console.log('myRole:', window.$nuxt?.$pinia?.gameStore?.myRole)
console.log('连接数量:', window.roomManager?.connections?.size)
console.log('所有连接:', window.roomManager?.connections)
```

**预期结果：**
- 主办方：`myRole: 'host'`，连接数量：2
- 选手1：`myRole: 'player'`，连接数量：1（连接到host）
- 选手2：`myRole: 'player'`，连接数量：1（连接到host）

### 3️⃣ 监控同步消息

在所有客户端的控制台中粘贴以下代码来监听同步消息：

```javascript
// 监控所有游戏状态更新
setInterval(() => {
  const store = window.$nuxt?.$pinia?.gameStore
  if (store) {
    console.log('[' + store.myRole + '] 状态版本:', store.stateVersion, '游戏状态:', store.gameStatus, '当前步骤:', store.currentRound?.step)
  }
}, 2000)
```

### 4️⃣ 测试同步

1. 在选手1的客户端上选择一个植物并点击"确认禁用"
2. 观察控制台输出

**预期看到的日志：**

在选手1客户端：
```
[gameStore] 已同步状态 v2
[RoomManager] 发送状态更新到主办方，版本: 2
```

在主办方客户端：
```
[RoomManager] 收到数据: stateUpdate
[gameStore] 收到状态更新 v2，来自 player
[gameStore] 准备转发状态 - myRole: host, senderRole: player
[gameStore] 执行转发状态 v2，排除 bp-room-xxxxx-player-xxxxx
```

在选手2客户端：
```
[RoomManager] 收到数据: stateUpdate
[gameStore] 收到状态更新 v2，来自 host
```

### 5️⃣ 常见问题排查

#### 问题A：看不到任何日志
**原因：** 控制台被过滤，或者日志级别设置不对
**解决：** 点击控制台左上角的过滤器，确保所有日志级别都启用

#### 问题B：myRole 显示 null 或 'multiplayer'
**原因：** 角色设置错误
**解决：** 检查 `src/store/gameStore.js` 中的 `setMyIdentity` 调用

#### 问题C：连接数量为0
**原因：** WebRTC 连接失败
**解决：**
1. 检查防火墙设置
2. 确认所有客户端都在同一网络
3. 检查 PeerJS 信令服务器是否可访问
4. 尝试刷新页面重新连接

#### 问题D：选手发送了消息，但主办方没收到
**原因：** 主办方的 PeerJS 连接处理有问题
**解决：** 检查 `src/utils/roomManager.js` 的 `handleIncomingConnection` 函数

#### 问题E：主办方收到了，但没有转发给选手2
**原因：** 转发逻辑问题，可能是 myRole 判断错误
**解决：** 检查控制台中"准备转发状态"日志，确认 myRole 是 'host'

### 6️⃣ 手动触发同步测试

在选手1的控制台中执行：

```javascript
// 手动触发同步
const store = window.$nuxt?.$pinia?.gameStore
if (store) {
  console.log('当前状态版本（同步前）:', store.stateVersion)
  store.syncState()
  setTimeout(() => {
    console.log('当前状态版本（同步后）:', store.stateVersion)
  }, 100)
}
```

然后观察其他客户端的控制台是否收到更新。

### 7️⃣ 检查 WebRTC 连接健康状态

```javascript
// 检查连接状态
const conn = window.roomManager?.connections?.values().next().value
if (conn) {
  console.log('连接状态:', {
    open: conn.open,
    peer: conn.peer,
    dataChannel: conn.dataChannel?.readyState
  })
}
```

**预期结果：** `open: true`，`dataChannel: 'open'`

## 快速修复尝试

如果以上诊断都无果，尝试以下修复步骤：

### 修复1：刷新所有客户端
1. 关闭所有三个浏览器窗口
2. 重新打开，重新创建房间和连接
3. 再次测试

### 修复2：清除浏览器缓存
```javascript
// 在控制台中执行
localStorage.clear()
location.reload()
```

### 修复3：检查网络环境
- 确保所有设备在同一网络
- 关闭VPN
- 检查防火墙是否阻止 WebRTC
- 尝试使用不同的浏览器（Chrome/Firefox/Edge）

### 修复4：更新代码
确保最新的修复已应用：
- `src/store/gameStore.js` 中的 `nextTick` 导入和使用
- `src/components/RoomSetup.vue` 中的 `setMyIdentity` 调用
